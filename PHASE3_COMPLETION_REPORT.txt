================================================================================
  三、Bug修复
================================================================================

Bug #1: 质量化类型值未保存
  发现: test_qualified_types() 测试失败
  症状: QINT/QREAL/QBOOL类型加载后值不正确
  原因: save函数的switch语句缺少质量化类型的case，导致value未写入文件
  修复: 在force_save_to_file()中为所有质量化类型添加case分支
  验证: 所有测试通过 ✅

================================================================================
  四、文件格式
================================================================================

示例配置文件 (test.force):
---
# STVM Force File
# Generated: 1761546794
force_enabled=1
force_count=3

force_var=temperature
type=qreal
value=25.500000
persistent=1

force_var=counter
type=qint
value=100
persistent=0

force_var=alarm
type=qbool
value=1
persistent=1
---

支持的类型:
  - int, real, bool, string (基础类型)
  - qint, qreal, qbool, qstring (质量化类型)

================================================================================
  六、使用示例
================================================================================

调试器中使用:

  # 强制几个变量
  (stvm-debug) force temperature 25.5
  (stvm-debug) force pressure 101.3
  (stvm-debug) force alarm true
  
  # 保存到文件
  (stvm-debug) force_save /tmp/my_config.force
  Force: Configuration saved to '/tmp/my_config.force' (3 entries)
  
  # 稍后加载
  (stvm-debug) force_load /tmp/my_config.force
  Force: Loaded 3 force variables from '/tmp/my_config.force'
  
  # 查看状态
  (stvm-debug) force_status

手动编辑配置文件:

  用户可以用文本编辑器直接编辑.force文件
  格式简单清晰，支持注释
  适合版本控制和团队共享
