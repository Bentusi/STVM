# ST语言编译器 - 第五阶段完成报告

## 概述

第五阶段成功实现了ST语言的运行时系统，包括虚拟机(VM)和库管理器(LibraryManager)。

## 实现内容

### 1. 虚拟机 (VM)

#### 1.1 数据结构
- **VM**: 虚拟机主结构
  - 操作数栈 (stack)
  - 调用栈 (call_stack)
  - 全局变量数组 (globals)
  - 程序计数器 (pc)
  - 运行状态和错误信息

- **CallFrame**: 调用帧结构
  - 返回地址 (return_address)
  - 基指针 (base_pointer)
  - 局部变量数量 (local_count)
  - 函数指针 (function)

#### 1.2 指令集实现
实现了完整的28条指令：

**栈操作指令**
- `OP_PUSH`: 将常量压入栈
- `OP_POP`: 弹出栈顶元素
- `OP_DUP`: 复制栈顶元素
- `OP_LOAD`: 加载变量（局部/全局）
- `OP_STORE`: 存储变量

**算术运算指令**
- `OP_ADD`: 加法
- `OP_SUB`: 减法
- `OP_MUL`: 乘法
- `OP_DIV`: 除法
- `OP_MOD`: 取模
- `OP_NEG`: 取负

**比较指令**
- `OP_EQ`: 等于
- `OP_NE`: 不等于
- `OP_LT`: 小于
- `OP_LE`: 小于等于
- `OP_GT`: 大于
- `OP_GE`: 大于等于

**逻辑运算指令**
- `OP_AND`: 逻辑与
- `OP_OR`: 逻辑或
- `OP_NOT`: 逻辑非

**控制流指令**
- `OP_JMP`: 无条件跳转
- `OP_JZ`: 条件跳转（为0跳转）
- `OP_JNZ`: 条件跳转（非0跳转）
- `OP_CALL`: 函数调用
- `OP_RET`: 函数返回

**其他指令**
- `OP_HALT`: 停机
- `OP_CALL_EXT`: 外部函数调用
- `OP_NOP`: 空操作

#### 1.3 核心功能
- **类型转换**: 支持整数和实数之间的自动转换
- **错误处理**: 栈溢出、栈下溢、除零、越界等错误检测
- **调试支持**: 虚拟机状态转储、调用栈追踪

#### 1.4 测试覆盖
- 基本算术运算
- 比较和跳转
- 全局变量
- 实数运算
- 栈操作
- 逻辑运算

## 关键问题解决

### 问题1: ConstantType 和 DataType 不匹配
**现象**: VM执行后栈上的值类型为TYPE_VOID

**原因**: 
```c
ConstantType: CONST_INT=0, CONST_REAL=1, CONST_BOOL=2, CONST_STRING=3
DataType:     TYPE_VOID=0, TYPE_BOOL=1, TYPE_INT=2, TYPE_REAL=3, ...
```

**解决**: 在OP_PUSH指令中正确映射ConstantType到DataType

### 问题2: 符号表字段名不一致
**现象**: 编译错误 - Symbol结构没有symbol_type字段

**解决**: 
- `symbol->symbol_type` → `symbol->kind`
- `symbol->type_info` → `symbol->type`

### 问题3: 函数定义参数类型缺失
**现象**: FunctionEntry中没有存储参数类型信息

**解决**: 临时方案 - 在构建符号表时假设所有参数为int类型
**建议**: 未来在字节码格式中包含完整的函数签名信息

### 问题4: OP_DUP宏的未定义行为
**现象**: 编译警告 - operation on 'vm->sp' may be undefined

**解决**: 使用临时变量避免嵌套的sp修改
