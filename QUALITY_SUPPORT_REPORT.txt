================================================================================
  一、需求背景
================================================================================

**用户需求**: "带质量位的变量不支持质量位强制"

**问题分析**:
  - 现有Force功能只能强制变量的值
  - 质量化类型（QINT, QREAL, QBOOL, QSTRING）的质量位无法被强制
  - 调试时无法模拟不同质量状态（GOOD, UNCERTAIN, BAD, ERROR）
  - 文件保存/加载时质量位信息丢失

**影响范围**:
  - 无法测试PLC质量位处理逻辑
  - 无法模拟传感器故障场景
  - 质量位配置无法持久化

================================================================================
  二、解决方案
================================================================================

### 2.1 核心API扩展

**新增函数 (types.c/types.h)**:

1. quality_to_string()
   - 将QualityFlag枚举转换为字符串
   - 支持: good, uncertain, bad, error

2. string_to_quality()
   - 将字符串解析为QualityFlag枚举
   - 支持字符串: good/uncertain/bad/error
   - 支持数字: 0/1/2/3
   - 容错：无效输入返回QUALITY_GOOD

### 2.2 Force功能增强

**force.c 修改**:

1. force_save_to_file() - 保存质量位
   - 对质量化类型自动添加quality=字段
   - 普通类型不保存quality字段

2. force_load_from_file() - 加载质量位
   - 解析quality=字段
   - 支持字符串和数字格式
   - 默认值为QUALITY_GOOD

3. force_print_status() - 显示质量位
   - 质量化类型显示: 值 [质量位]
   - 示例: "100 [bad]", "25.5 [uncertain]"

### 2.3 调试器命令增强

**debugger.c 修改**:

1. cmd_force() - 支持质量位参数
   语法: force <var> <type> <value> [quality] [persistent]
   
   示例:
     force sensor qint 100 bad
     force temp qreal 25.5 uncertain persistent

2. cmd_force_status() - 显示质量位
   输出格式:
     sensor1    qint    999 [bad]           NO
     temp       qreal   25.500000 [uncertain]  YES

### 2.4 文件格式扩展

**新格式**:
```
force_var=sensor
type=qint
quality=bad          # 新增字段（仅质量化类型）
value=999
persistent=0
```

**向后兼容**:
- 旧文件没有quality字段 → 自动设为QUALITY_GOOD
- 普通类型不保存quality字段
- 现有配置文件继续有效

### 关键实现

**1. 质量位转换 (types.c)**:
```c
const char* quality_to_string(QualityFlag quality) {
    switch (quality) {
        case QUALITY_GOOD:      return "good";
        case QUALITY_UNCERTAIN: return "uncertain";
        case QUALITY_BAD:       return "bad";
        case QUALITY_ERROR:     return "error";
        default:                return "unknown";
    }
}

QualityFlag string_to_quality(const char* str) {
    if (!str) return QUALITY_GOOD;
    if (strcmp(str, "good") == 0 || strcmp(str, "0") == 0) 
        return QUALITY_GOOD;
    // ... 其他映射
    return QUALITY_GOOD;  // 默认值
}
```

**2. 文件保存质量位 (force.c)**:
```c
// 对于质量化类型，保存质量位
switch (current->forced_value.type) {
    case TYPE_QINT:
    case TYPE_QREAL:
    case TYPE_QBOOL:
    case TYPE_QSTRING:
        fprintf(f, "quality=%s\n", 
                quality_to_string(current->forced_value.quality));
        break;
    default:
        break;  // 普通类型不保存quality
}
```

**3. 文件加载质量位 (force.c)**:
```c
else if (strcmp(key, "quality") == 0) {
    // 解析质量位（仅对质量化类型有效）
    value.quality = string_to_quality(val);
}
```

**4. 调试器命令解析 (debugger.c)**:
```c
// 解析可选参数（质量位和persistent）
while (next_arg) {
    if (strcasecmp(next_arg, "persistent") == 0) {
        persistent = true;
    } else if (strcasecmp(next_arg, "bad") == 0) {
        quality = QUALITY_BAD;
    }
    // ... 其他质量位
    next_arg = strtok(NULL, " \t");
}

// 对于质量化类型，设置质量位
if (value.type == TYPE_QINT || value.type == TYPE_QREAL || 
    value.type == TYPE_QBOOL || value.type == TYPE_QSTRING) {
    value.quality = quality;
}
```

  五、使用示例
================================================================================

### 示例1: 调试器中使用质量位

```bash
# 启动调试器
./build/bin/stvm -d program.stbc

# 强制传感器为坏值
(stvm-debug) force sensor1 qreal 0.0 bad persistent

# 强制传感器为不确定值
(stvm-debug) force sensor2 qreal 25.5 uncertain

# 强制传感器为错误状态
(stvm-debug) force sensor3 qreal 0.0 error

# 查看状态
(stvm-debug) force_status

输出:
=== Force Manager Status ===
Force Enabled: YES
Force Count: 3

Forced Variables:
Name/Index                     Type       Forced Value         Persistent
----------                     ----       ------------         ----------
sensor1                        qreal      0.000000 [bad]       YES
sensor2                        qreal      25.500000 [uncertain] NO
sensor3                        qreal      0.000000 [error]     NO
```

### 示例2: 配置文件

**创建故障场景配置 (fault.force)**:
```
# 传感器故障场景
force_enabled=1
force_count=3

force_var=AI_temperature
type=qreal
quality=bad
value=0.0
persistent=1

force_var=AI_pressure
type=qreal
quality=uncertain
value=100.0
persistent=1

force_var=DI_alarm
type=qbool
quality=error
value=1
persistent=0
```

**在调试器中加载**:
```bash
(stvm-debug) force_load fault.force
Force: Loaded 3 force variables from 'fault.force'

(stvm-debug) force_status
# 显示所有强制变量及其质量位
```

  六、兼容性
================================================================================

### 向后兼容性 ✅

1. **现有Force配置文件**
   - 旧文件无quality字段 → 自动使用QUALITY_GOOD
   - 新代码可以加载旧文件
   - 旧代码无法解析质量位（但不会崩溃）

2. **普通类型不受影响**
   - INT, REAL, BOOL, STRING类型行为不变
   - 质量位参数对普通类型无效（被忽略）

3. **API兼容**
   - force_variable()接口不变
   - 新增的quality_to_string()等函数是额外的

### 文件格式演进

**v1.0 (Phase 3)** - 基础文件格式:
```
force_var=sensor
type=qint
value=100
persistent=0
```

**v1.1 (Phase 3.1)** - 质量位支持:
```
force_var=sensor
type=qint
quality=bad      # 新增字段
value=100
persistent=0
```

**兼容性**: v1.1可以读取v1.0文件 ✅
